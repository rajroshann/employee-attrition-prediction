# 🧠 Employee Attrition Prediction

An end-to-end machine learning web application that predicts whether an employee
will leave a company — and explains **why**, using SHAP-based Explainable AI.

Built on the [IBM HR Analytics Employee Attrition Dataset](https://www.kaggle.com/datasets/pavansubhasht/ibm-hr-analytics-attrition-dataset).

## 🗂️ Project Modules

| Module | What it does |
|--------|-------------|
| **Module 1 — Dashboard** | KPI cards: total employees, attrition %, avg income, age, experience, gender ratio |


Open [http://localhost:5000](http://localhost:5000) in your browser.

## 🧪 ML Pipeline

### Dataset
- **Source:** IBM HR Analytics (Kaggle)
- **Rows:** 1,470 employees
- **Target:** `Attrition` (Yes/No → 1/0)
- **Class imbalance:** 83.9% No / 16.1% Yes

### Preprocessing
| Column type | Treatment |
|-------------|-----------|
| Noise columns (`EmployeeCount`, `EmployeeNumber`, `Over18`, `StandardHours`) | Dropped |
| Binary categorical (`Gender`, `OverTime`) | Label Encoding |
| Multi-class categorical (`Department`, `JobRole`, `MaritalStatus`, etc.) | One-Hot Encoding |
| Numerical (23 columns) | StandardScaler |

All steps bundled in a single `sklearn Pipeline` — no leakage, no manual transforms at inference.

### Model: XGBoost Classifier
Chosen over Logistic Regression and Random Forest because:
- Best performance on mixed-type tabular data
- Native class imbalance handling via `scale_pos_weight`
- SHAP `TreeExplainer` is exact (not approximate) for tree-based models

### Class Imbalance Handling
```python
scale_pos_weight = count(No) / count(Yes)  # ≈ 5.2
```
Penalises the model more for missing `Yes` predictions — the ones that actually cost the business.

### Evaluation Metrics
| Metric | Score | Why it matters |
|--------|-------|----------------|
| Recall (Leave=1) | 0.36 | Catching leavers is the primary goal — a missed leaver is a real cost |
| F1 Score | 0.49 | Balances precision and recall — more honest than accuracy on imbalanced data |
| ROC-AUC | 0.76 | Model ranks leavers above stayers 76% of the time |
| Accuracy | 0.86 | Reported last — inflated by the majority class |
| 5-Fold CV F1 | 0.49 ± 0.04 | Confirms the model generalises, not just lucky on one split |

## 🔍 Explainable AI (SHAP)

SHAP (SHapley Additive exPlanations) answers:
> _"For this specific employee, which features pushed the prediction toward leaving — and by how much?"_

```
Base rate (16.1%)
  + OverTime = Yes        → +0.18 push
  + JobSatisfaction = Low → +0.12 push
  + MonthlyIncome = 2500  → +0.09 push
  = 87% predicted attrition
```

`TreeExplainer` is used (not `KernelExplainer`) because it is:
- **Exact** — not an approximation
- **Fast** — O(T × L × D) vs O(N²) for kernel
- **Native** — built for tree-based models like XGBoost

## 🚀 Deployment (Render)


- Start command: `gunicorn app:app`
